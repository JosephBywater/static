Why do I need an API key?

It was a tough decision to make, but in May 2016 we decided to require an API key for proper access to our APIs.

We had to do this for a number of reasons:

  • To ease communication between us and you.
  • To allow us to tailor the API to your needs.
  • To make it easier to block malicious users.
  • To ensure fair use of the API.

Am I rate limited for cached responses?

If our servers have a response for your request cached then no, it will not be rate limited nor will you require an API key.

In other words: If you are asking for something that is cached you don't need an API key and rate limits don't apply!

This may seem silly, but there are some niche use cases for this functionality, it also speeds up cached responses.

Why do I need to login with Twitter?

We use Twitter to handle authentication, this allows you to have all of the security features built in to Twitter (such as two-factor authentication) as well as the convenience of not needing a username/password for another site.

We cannot post tweets, access your direct messages or other private information.
