Player Username to UUID example

An example of the Username to UUID api can be found below.

Basic Lookup

Example of getting a UUID, 'Notch' as the example:

https://#HOST#/v3/uuid/notch

Queries are not case sensitive, therefore you don't need to worry about capital letters to find a uuid. This query will return a json object like so:

{
    "full_uuid" : "069a79f4-44e9-4726-a5be-fca90e38aaf5",
    "source" : "mojang",
    "query" : "notch",
    "name" : "Notch",
    "took" : 24,
    "uuid" : "069a79f444e94726a5befca90e38aaf5"
}


This object conatins the UUID associated with the query username, along with the case-correct username. If the username is not registed with mojang, the following will be returned:
{"error":"Unknown Username","query":"notch"}

All queries that go to mojang will take slightly longer than others.

Field Object Description
"full_uuid"StringUUIDThe uuid of the player in java format.
"source"StringIf the server looked in the database, cache or mojang.
"query"StringThe original query.
"name"StringThe username in correct case.
"took"IntegerHow long the query took.
"uuid"StringThe uuid without "-"'s

Advanced Usage

You can request multiple uuids at the same time, example using 'Dinnerbone' and 'Notch':

https://#HOST#/v3/uuid/notch,dinnerbone

You can request upto 15 uuids at a time. Any failed requests will be in the failed array

{
    "failed": [],
    "success": [
        {
            "full_uuid": "069a79f4-44e9-4726-a5be-fca90e38aaf5",
            "source": "database",
            "query": "notch",
            "name": "Notch",
            "took": 241,
            "uuid": "069a79f444e94726a5befca90e38aaf5"
        },
        {
            "full_uuid": "61699b2e-d327-4a01-9f1e-0ea8c3f06bc6",
            "source": "database",
            "query": "dinnerbone",
            "name": "Dinnerbone",
            "took": 41,
            "uuid": "61699b2ed3274a019f1e0ea8c3f06bc6"
        }
    ]
}
